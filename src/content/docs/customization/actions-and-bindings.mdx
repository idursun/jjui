---
title: Actions and Bindings
description: Define custom actions and key bindings in config.toml or config.lua
---

Actions are named operations. Bindings connect actions to keys. Both can be defined in `config.toml` or `config.lua` — use whichever fits your workflow.

## Defining an action

### In config.toml

Use `[[actions]]` with an inline Lua snippet:

```toml
[[actions]]
name = "copy-change-id"
desc = "copy change id to clipboard"
lua = '''
local id = context.change_id()
if id then
  copy_to_clipboard(id)
  flash("Copied: " .. id)
end
'''
```

### In config.lua

Pass a function directly to `config.action`:

```lua
function setup(config)
  config.action("copy-change-id", function()
    local id = context.change_id()
    if id then
      copy_to_clipboard(id)
      flash("Copied: " .. id)
    end
  end, { desc = "copy change id to clipboard" })
end
```

## Binding an action to a key

### In config.toml

```toml
[[bindings]]
action = "copy-change-id"
key = "Y"
scope = "revisions"
desc = "copy change id"
```

### In config.lua

Either pass binding options directly to `config.action`:

```lua
config.action("copy-change-id", fn, {
  key = "Y",
  scope = "revisions",
  desc = "copy change id",
})
```

Or bind separately with `config.bind`:

```lua
config.bind({
  action = "copy-change-id",
  key = "Y",
  scope = "revisions",
  desc = "copy change id",
})
```

## Binding fields

| Field | Required | Notes |
|-------|----------|-------|
| `action` | yes | Action name or built-in action ID |
| `scope` | yes | Where the binding is active |
| `key` or `seq` | yes | Set exactly one |
| `desc` | no | Label shown in the help/status bar |
| `args` | no | Arguments for built-in actions |

`key` accepts a single key or a list of alternatives:

```toml
key = "r"
key = ["up", "k"]
```

`seq` defines an ordered key sequence (chord):

```toml
[[bindings]]
action = "ui.open_revset"
seq = ["g", "r"]
scope = "revisions"
desc = "open revset"
```

## Rebinding built-in actions

You can bind any built-in action by its action ID:

```toml
[[bindings]]
action = "ui.open_revset"
key = "L"
scope = "revisions"
desc = "revset"
```

All built-in action IDs and their default bindings are in `internal/config/default/bindings.toml`. See [Default Keybindings](/jjui/customization/default-keybindings) for the full reference.

---

## Overriding built-in actions

You can replace a built-in action's behavior entirely. Call the original via `jjui.builtin.*` to preserve it:

```lua
config.action("revisions.toggle_select", function()
  jjui.builtin.revisions.toggle_select()
  jjui.builtin.revisions.move_down()
end)
```

This overrides `toggle_select` to also advance the cursor after selecting.
